## Design Decisions

### Decision 1: 删除所有推理系统

**理由**：
- 推理结果无法验证（POB 模拟数据无法还原游戏逻辑）
- 312 条 bypass 记录全部未验证通过
- 维护成本高，实际价值低

**影响**：
- 删除约 10,000 行代码
- 节省约 13.5MB 存储空间
- 大幅降低维护成本

### Decision 2: 保留静态数据系统

**理由**：
- `entities.db`：16,461 个实体，可靠的静态数据
- `formulas.db`：1,486 个公式，直接从 POB 代码提取
- `mechanisms.db`：44 个机制描述，可选保留

**影响**：
- 保留核心价值：数据查询和公式查询
- 用户可以查询任何实体属性
- 用户可以查询任何 stat 的计算公式

### Decision 3: 简化初始化流程

**理由**：
- 从 7 步减少到 4 步
- 删除规则提取、关联图构建步骤
- 保留数据提取、索引构建步骤

**影响**：
- 初始化更快（预计减少 50% 时间）
- 流程更简单，更容易维护

### Decision 4: 简化查询接口

**理由**：
- 删除规则查询、图查询
- 保留实体查询、公式查询
- 删除复杂的查询路由逻辑

**影响**：
- 查询接口更直接
- 用户不需要理解"三问法"
- 查询性能更好

## Implementation Strategy

### Phase 1: 删除数据库和配置（安全清理）

1. 备份现有知识库
2. 删除 `graph.db`、`rules.db`
3. 删除 `predefined_edges.yaml`、`edge_semantics.yaml`、`rule_templates.yaml`
4. 删除 `config/mechanisms/` 目录

### Phase 2: 删除脚本文件（核心清理）

1. 删除关联图系统：`attribute_graph.py`
2. 删除规则系统：`rules_extractor.py`
3. 删除验证系统：`verification/` 目录、相关脚本
4. 删除启发推理系统：`heuristic_*.py`
5. 删除约束系统：`constraint_*.py`、`tag_source_finder.py`
6. 删除查询引擎：`query_router.py`、`query_engine.py`
7. 删除临时脚本：`analyze_*.py`、`check_*.py`、`verify_*.py`

### Phase 3: 删除文档目录（文档清理）

1. 删除 `docs/` 目录（17 个文件）
2. 删除 `exploration/` 目录

### Phase 4: 修改核心文件（功能调整）

1. 修改 `init_knowledge_base.py`：
   - 删除 Step 4（规则提取）
   - 删除 Step 5（关联图构建）
   - 简化 Step 7（Schema 验证）

2. 修改 `kb_query.py`：
   - 删除 `get_rule()` 方法
   - 删除 `get_graph_neighbors()` 方法
   - 删除 `find_bypass_path()` 方法
   - 保留实体和公式查询方法

3. 重写 `SKILL.md`：
   - 删除推理相关章节
   - 更新为"数据问答服务"定位
   - 明确能力边界

4. 清理 `schemas.json`：
   - 删除 `rules` 表定义
   - 删除 `graph_nodes` 表定义
   - 删除 `graph_edges` 表定义
   - 删除 `constraints` 表定义

### Phase 5: 测试和验证（质量保证）

1. 运行初始化脚本，确保 4 步流程正常
2. 测试实体查询功能
3. 测试公式查询功能
4. 检查无遗留的 import 错误

## Risks and Mitigations

### Risk 1: 用户期望错位

**风险**：用户习惯了推理功能，突然删除可能引起困惑

**缓解**：
- 在 SKILL.md 中明确说明能力边界
- 提供"能做什么"和"不能做什么"的清晰说明
- 在查询结果中添加提示信息

### Risk 2: 数据完整性

**风险**：删除过程中可能误删重要数据

**缓解**：
- 先备份整个 `knowledge_base/` 目录
- 先备份 `schemas/schemas.json`
- 分阶段删除，每阶段验证

### Risk 3: 代码依赖

**风险**：其他文件可能依赖被删除的模块

**缓解**：
- 使用 Schema 管理系统检查依赖
- 删除前搜索 import 语句
- 删除后运行测试套件
